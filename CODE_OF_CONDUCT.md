# Code Of Conduct

## Our Standard

Contributors are expected to be respectful, direct, and technically honest.

We want a project environment where people can:

- ask hard questions
- challenge ideas with evidence
- review code rigorously
- collaborate without harassment or hostility

## Unacceptable Behavior

- harassment or personal attacks
- doxxing or sharing private credentials
- abusive or discriminatory language
- intentionally misleading technical claims
- publishing exploit details irresponsibly

## Enforcement

Project maintainers may remove content, restrict access, or ban contributors whose behavior harms the project or its participants.
